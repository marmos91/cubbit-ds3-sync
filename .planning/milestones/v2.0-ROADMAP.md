# Milestone v2.0: iOS & iPadOS Universal App

**Status:** SHIPPED 2026-03-20
**Phases:** 6-9
**Total Plans:** 17

## Overview

DS3 Drive works on iPhone and iPad with the same reliability as macOS -- users can log in, set up drives, and access their S3 files through the native Files app. Includes Share Extension for uploading from other apps and sync badges in Files app.

## Milestone Summary

**Key Decisions:**
- Protocol abstraction pattern (IPCService, SystemService, LifecycleService) for cross-platform shared code
- Darwin notifications + App Group file payloads for iOS IPC (vs DistributedNotificationCenter on macOS)
- Streaming I/O for iOS extension memory safety (zero-copy ByteBuffer writes)
- Cache-first + 60s TTL enumeration pattern for both platforms
- Sequential file uploads in Share Extension to conserve memory
- Mirrored design tokens in Share Extension (separate from main app target)

**Issues Resolved:**
- iOS extension memory constraints (streaming I/O, adaptive fetch semaphore)
- Cross-platform IPC (Darwin notifications vs DistributedNotificationCenter)
- Background polling disabled on iOS (enumeration-time checks instead)
- Share Extension UI lifecycle bridging (SwiftUI inside UIHostingController)

**Technical Debt Incurred:**
- Phase 5 (v1.0 UX Polish) still has 2 unexecuted plans (05-04, 05-05)
- ROADMAP checkboxes for phases 6, 7, 9 were not fully updated during execution
- FOUN-04 (SwiftData metadata database) still pending from v1.0
