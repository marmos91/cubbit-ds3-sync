<objective>
Create the IPCService protocol and both platform implementations (macOS + iOS) for inter-process communication between the main app and File Provider extension.

Purpose: IPCService is the core abstraction for ABST-01. It replaces all direct DistributedNotificationCenter usage with a protocol that both macOS and iOS can implement. The macOS implementation wraps existing DistributedNotificationCenter in AsyncStream; the iOS implementation uses Darwin notifications for signaling combined with App Group file payloads for data exchange. Creating iOS-compatible abstractions with #if os() guards directly contributes to ABST-04.

Output: IPCService protocol, MacOSIPCService, IOSIPCService, DarwinNotificationCenter wrapper, IPCCommand enum, and unit tests.

**3. `DarwinNotificationCenter.swift`** -- Swift wrapper around CFNotificationCenterGetDarwinNotifyCenter (NO `#if os()` guard -- Darwin notifications work on both macOS and iOS):
- `public final class DarwinNotificationCenter: Sendable`
- `public static let shared = DarwinNotificationCenter()`
- `private let center: CFNotificationCenter` initialized with `CFNotificationCenterGetDarwinNotifyCenter()`
- `public func post(name: String)` -- posts via `CFNotificationCenterPostNotification`
- `public func addObserver(name: String, callback: @escaping @Sendable () -> Void) -> DarwinNotificationObservation` -- registers C callback via `Unmanaged<Box>` pattern
- `public func notifications(named name: String) -> AsyncStream<Void>` -- returns AsyncStream, uses `addObserver` internally, cancels on `onTermination`
- `public final class DarwinNotificationObservation: @unchecked Sendable` with `cancel()` method and `deinit` that calls `cancel()`
  - Uses a private `Box` class to bridge the C callback: store the Swift callback in a Box, pass `Unmanaged.passRetained(box).toOpaque()` as the observer pointer, retrieve it in the C callback via `Unmanaged<Box>.fromOpaque(pointer).takeUnretainedValue()`
  - `cancel()` calls `CFNotificationCenterRemoveObserver` and then `Unmanaged.fromOpaque(pointer).release()` to balance the retain

IMPORTANT: The C callback signature for `CFNotificationCenterAddObserver` is:
```c
@convention(c) (CFNotificationCenter?, UnsafeMutableRawPointer?, CFNotificationName?, UnsafeRawPointer?, CFDictionary?) -> Void
```
The `UnsafeMutableRawPointer?` is the observer pointer we pass. Use this to retrieve the Box.

All files must compile under Swift 6 concurrency. Use `@unchecked Sendable` where the C bridge requires it but the implementation is thread-safe.

**2. `IPCService+iOS.swift`** -- Entire file wrapped in `#if os(iOS)`:
- `final class IOSIPCService: IPCService, @unchecked Sendable`
- Store same continuation pattern as macOS
- `private let ipcDirectory: URL` -- `FileManager.default.containerURL(forSecurityApplicationGroupIdentifier: DefaultSettings.appGroup)!.appendingPathComponent("ipc", isDirectory: true)`. Create directory in init if it doesn't exist.
- Post methods follow the pattern:
  1. JSON-encode the payload
  2. Write to a temp file in ipcDirectory: `ipcDirectory.appendingPathComponent(UUID().uuidString + ".tmp")`
  3. Atomic rename to target file: `FileManager.default.replaceItemAt(targetURL, withItemAt: tmpURL)` (or write then moveItem). Target filenames: `statusChange.json`, `transferStats.json`, `command.json`, `conflict.json`, `authFailure.json`, `extensionInitFailure.json`
  4. Post Darwin notification via `DarwinNotificationCenter.shared.post(name:)` using the same DefaultSettings.Notifications name strings
- `startListening()`:
  - For each channel, use `DarwinNotificationCenter.shared.notifications(named:)` to get an `AsyncStream<Void>`
  - Spin up a Task that iterates over the stream, and on each signal reads the corresponding JSON file from ipcDirectory, decodes it, and yields to the continuation
  - Additionally, start a polling task that runs every 30 seconds, reads all IPC files, and yields any changes (safety net per user decision for missed Darwin notifications)
  - Store all tasks for cleanup
- `stopListening()`:
  - Cancel all listener tasks and polling task
  - Finish all continuations
- Atomic write helper (private):
  ```swift
  private func writeAtomically<T: Encodable>(_ value: T, toFile filename: String) {
      let targetURL = ipcDirectory.appendingPathComponent(filename)
      let tmpURL = ipcDirectory.appendingPathComponent(UUID().uuidString + ".tmp")
      guard let data = try? JSONEncoder().encode(value) else { return }
      do {
          try data.write(to: tmpURL)
          _ = try? FileManager.default.removeItem(at: targetURL)
          try FileManager.default.moveItem(at: tmpURL, to: targetURL)
      } catch {
          try? FileManager.default.removeItem(at: tmpURL)
      }
  }
  ```

Both implementations must be `@unchecked Sendable` and compile under Swift 6. All DistributedNotificationCenter calls are ONLY in the macOS file behind `#if os(macOS)`.
