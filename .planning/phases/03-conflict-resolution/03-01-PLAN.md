<feature>
  <name>ConflictNaming utility</name>
  <files>DS3Lib/Sources/DS3Lib/Utils/ConflictNaming.swift, DS3Lib/Tests/DS3LibTests/ConflictNamingTests.swift</files>
  <behavior>
    - conflictKey("photos/report.pdf", hostname: "amaterasu", date: 2024-01-15 14:30:45) -> "photos/report (Conflict on amaterasu 2024-01-15 14-30-45).pdf"
    - conflictKey("README", hostname: "mac", date: ...) -> "README (Conflict on mac YYYY-MM-DD HH-MM-SS)" (no extension)
    - conflictKey("docs/my.file.txt", hostname: "mac", date: ...) -> "docs/my.file (Conflict on mac YYYY-MM-DD HH-MM-SS).txt" (dots in name, only last dot is extension split)
    - conflictKey("report.pdf", hostname: "mac", date: ...) -> "report (Conflict on mac YYYY-MM-DD HH-MM-SS).pdf" (root-level file, no parent path)
    - conflictKey("a/b/c/deep.jpg", hostname: "mac", date: ...) -> "a/b/c/deep (Conflict on mac YYYY-MM-DD HH-MM-SS).jpg" (deeply nested path preserved)
    - conflictKey("unicode-file-emojis.txt", hostname: "Marcos-MacBook", date: ...) -> preserves unicode correctly
  </behavior>
  <implementation>
    Create `enum ConflictNaming` in DS3Lib/Sources/DS3Lib/Utils/ConflictNaming.swift.
    Single static method `conflictKey(originalKey:hostname:date:)` that:
    1. Splits key by "/" to extract parent path and filename
    2. Splits filename by last "." to separate name and extension
    3. Formats date with DateFormatter using "yyyy-MM-dd HH-mm-ss" format
    4. Assembles: "\(name) (Conflict on \(hostname) \(dateStr))\(ext)" where ext includes the leading dot
    5. Reconstructs full key with parent path prefix
    Must be public, Sendable-compatible (enum with static method = naturally Sendable).
  </implementation>
</feature>

<feature>
  <name>ETagUtils utility</name>
  <files>DS3Lib/Sources/DS3Lib/Utils/ETagUtils.swift, DS3Lib/Tests/DS3LibTests/ETagUtilsTests.swift</files>
  <behavior>
    - normalize("\"abc123\"") -> "abc123" (strips surrounding quotes)
    - normalize("abc123") -> "abc123" (no quotes, unchanged)
    - normalize("") -> "" (empty string)
    - normalize(nil) -> nil (nil passthrough)
    - areEqual("\"abc123\"", "abc123") -> true (normalized comparison)
    - areEqual("abc", "def") -> false
    - areEqual(nil, "abc") -> false (nil never equals anything)
    - areEqual(nil, nil) -> false (nil never equals nil -- we need both ETags to exist for a valid comparison)
  </behavior>
  <implementation>
    Create `enum ETagUtils` in DS3Lib/Sources/DS3Lib/Utils/ETagUtils.swift.
    Two static methods:
    - `normalize(_ etag: String?) -> String?` strips surrounding double-quotes if present
    - `areEqual(_ lhs: String?, _ rhs: String?) -> Bool` normalizes both sides and compares; returns false if either is nil
    Must be public and Sendable-compatible.
  </implementation>
</feature>
