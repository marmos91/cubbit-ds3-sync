    **S3Item+Metadata.swift:**
    - Add `var syncStatus: String?` to S3Item.Metadata if not already present (String to match MetadataStore's String-based storage)
    - This field is populated when constructing S3Item from MetadataStore query results

    **S3Item.swift:**
    - Add `NSFileProviderItemDecorating` conformance to S3Item class declaration
    - Define static decoration identifiers using `decorationPrefix`:
      ```swift
      static let decorationSynced = NSFileProviderItemDecorationIdentifier(rawValue: "\(decorationPrefix).synced")
      static let decorationSyncing = NSFileProviderItemDecorationIdentifier(rawValue: "\(decorationPrefix).syncing")
      static let decorationError = NSFileProviderItemDecorationIdentifier(rawValue: "\(decorationPrefix).error")
      static let decorationCloudOnly = NSFileProviderItemDecorationIdentifier(rawValue: "\(decorationPrefix).cloudOnly")
      static let decorationConflict = NSFileProviderItemDecorationIdentifier(rawValue: "\(decorationPrefix).conflict")
      ```
    - Implement `var decorations: [NSFileProviderItemDecorationIdentifier]?` computed property
    - Map metadata.syncStatus string to decoration identifier:
      - "synced" -> decorationSynced
      - "syncing" -> decorationSyncing
      - "error" -> decorationError
      - "conflict" -> decorationConflict
      - "pending" or nil -> decorationCloudOnly (pending items are cloud-only)
    - For folders (isFolder == true): if metadata.syncStatus is nil, return decorationCloudOnly
    - IMPORTANT: The identifier strings MUST exactly match the Info.plist keys (use decorationPrefix which is the extension bundle identifier)

    **Info.plist:**
    - Add `NSFileProviderDecorations` dictionary inside the existing `NSExtension` dictionary
    - Add 5 decoration entries, each keyed by the full identifier (e.g., `io.cubbit.DS3Drive.DS3DriveProvider.synced`):
      - synced: BadgeImageType=Symbol, SymbolName=checkmark.circle.fill, Label=Synced, Category=SyncStatus
      - syncing: BadgeImageType=Symbol, SymbolName=arrow.triangle.2.circlepath, Label=Syncing, Category=SyncStatus
      - error: BadgeImageType=Symbol, SymbolName=xmark.circle.fill, Label=Error, Category=SyncStatus
      - cloudOnly: BadgeImageType=Symbol, SymbolName=cloud.fill, Label=Cloud Only, Category=SyncStatus
      - conflict: BadgeImageType=Symbol, SymbolName=exclamationmark.triangle.fill, Label=Conflict, Category=SyncStatus
    - The decoration identifier prefix must match `Bundle.main.bundleIdentifier` which for the extension is `io.cubbit.DS3Drive.DS3DriveProvider`
    - Place NSFileProviderDecorations INSIDE the NSExtension dict (not at the top level)

    **Folder badge aggregation:**
    - For now, folders use their own syncStatus from metadata (set during enumeration). The SyncEngine already computes folder status during reconciliation.
    - If a folder has no explicit syncStatus, default to cloudOnly.
    - Deep aggregation (querying all children) is deferred to avoid performance issues per RESEARCH.md recommendation.

    NOTE: S3Item does NOT need to import DS3Lib for SyncStatus enum -- it just uses the raw String values from metadata.syncStatus for comparison. This avoids adding DS3Lib as a dependency to the pattern-matching logic.
  </action>
