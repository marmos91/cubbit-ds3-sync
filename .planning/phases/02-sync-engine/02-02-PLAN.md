**Replace the minimal stub SyncEngine.swift** with the full implementation:

Design decisions per CONTEXT.md locked decisions:
- Full reconciliation on every call (S3 listObjectsV2 vs MetadataStore diff)
- Hard delete SyncedItem records when remote deletion detected
- Threshold warning at >50% deletions (log but proceed)
- 3 consecutive failures = error state
- NetworkMonitor check before operations
- SyncEngine owns ALL MetadataStore writes

SyncEngine actor implementation:
- Properties: `metadataStore: MetadataStore`, `networkMonitor: NetworkMonitor`, `logger`, weak `delegate: SyncEngineDelegate?`
- `init(metadataStore: MetadataStore, networkMonitor: NetworkMonitor)`
- `func setDelegate(_ delegate: SyncEngineDelegate)` (actor method to set delegate)
- `func reconcile(driveId: UUID, s3Provider: S3ListingProvider, bucket: String, prefix: String?) async throws -> ReconciliationResult`
  1. Check `networkMonitor.isConnected` -- throw `NSFileProviderError(.serverUnreachable)` if offline
  2. Fetch all remote items via `s3Provider.listAllItems(bucket:prefix:)`
  3. Fetch all local items via `metadataStore.fetchItemsByDrive(driveId:)`
  4. Build sets: remoteKeySet, localKeySet
  5. Compute: newKeys = remote - local, deletedKeys = local - remote (only where syncStatus == .synced), commonKeys = intersection
  6. For common keys, compare etags to find modifiedKeys
  7. Check mass deletion threshold (deletedKeys.count > localItems.count / 2), log warning if true
  8. Upsert new + modified items into MetadataStore with syncStatus .synced
  9. Delete removed items from MetadataStore (hard delete per decision)
  10. Advance sync anchor via `metadataStore.advanceSyncAnchor(driveId:itemCount:)`
  11. Reset failure count
  12. Notify delegate of completion
  13. Return ReconciliationResult
- On catch: increment failure count via MetadataStore, if >= 3 notify delegate of error state, re-throw
